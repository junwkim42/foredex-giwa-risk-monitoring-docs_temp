[ForeDex GIWA Risk Monitoring 테스트 대시보드 열기](https://test.foredex.io/charts/altcoin/giwa-risk-monitoring)

## 화면의 역할

대시보드는 서로 다른 온체인 데이터를 프로젝트 단위로 연결하고, 설정된 규칙에 해당하는 변화를 검토 가능한 형태로 요약합니다. 화면은 조사 시작점을 제공하며 Explorer의 원시 기록을 대체하지 않습니다.

## 권장 확인 순서

1. **환경 확인:** GIWA Sepolia와 demo-only 표시를 확인합니다.
2. **대상 확인:** 토큰 컨트랙트, 프로젝트 지갑, DEX 풀을 확인합니다.
3. **기준 확인:** 공시값, 임계값과 시간창을 확인합니다.
4. **관측값 확인:** 공급량, 민팅, 순유출과 스왑 값을 확인합니다.
5. **상태 이유 확인:** 어떤 데이터가 규칙 조건을 충족했는지 확인합니다.
6. **Explorer 이동:** 컨트랙트·이벤트·트랜잭션 링크로 근거를 재검증합니다.

## 상태 표기의 의미

| 표기 | 의미 |
|---|---|
| NORMAL | 현재 관측값이 설정된 WARNING·CRITICAL 조건에 해당하지 않음 |
| WARNING | 예시 경고 임계값에 해당해 우선 검토가 필요함 |
| CRITICAL | 예시 상위 임계값 또는 max supply 조건에 해당해 높은 우선순위로 검토가 필요함 |
| CONFIRMED | 해당 블록·로그를 기준으로 위험 이벤트 레코드가 확정됨 |

`CONFIRMED`는 시스템 이벤트 상태이며 프로젝트의 위법행위, 고의 또는 실제 손실이 확인됐다는 의미가 아닙니다.

## 동적 값에 대한 주의

데모를 반복 실행하면 공급량, 지갑 잔액, 스왑과 알림 수가 변할 수 있습니다. 화면 수치를 문서나 심사 자료에 인용할 때는 캡처 시각과 블록을 함께 표시해야 합니다. 초기 데모 설정값과 현재 온체인 관측값을 같은 값으로 취급하지 않습니다.

## 상태가 말하지 않는 것

- 경고가 없다는 사실이 위험이 없음을 보장하지 않습니다.
- 경고가 발생했다는 사실이 고의, 범죄, 부실 또는 투자 손실을 증명하지 않습니다.
- 화면의 우선순위는 설정된 주소·풀·규칙·임계값 범위에서만 유효합니다.
- 데이터 인덱싱과 네트워크 상태에 따라 화면과 최신 블록 사이에 지연이 있을 수 있습니다.

## 갱신 방식

대시보드의 기본 데이터 요청은 60초 간격으로 갱신되고, 새로운 위험 이벤트는 별도 이벤트 스트림으로 전달되도록 구성되어 있습니다. 60초는 화면의 기본 조회 주기이며 수집부터 판정·표시까지의 종단 간 탐지 지연 보장값이 아닙니다. 따라서 본 문서는 `실시간` 대신 `주기적 모니터링`과 `이벤트 스트림`으로 표현합니다.

# 시스템 및 데이터 흐름

## 처리 흐름

1. **모니터링 대상 정의:** 프로젝트 관련 지갑, 토큰 컨트랙트, DEX 풀과 비교 기준을 등록합니다.
2. **온체인 데이터 수집:** 잔액, 전송, 민팅, 총공급량, 스왑과 유동성 관련 데이터를 수집합니다.
3. **정규화 및 연계:** 주소·토큰·풀 데이터를 프로젝트 단위로 연결하고 비교 가능한 형태로 변환합니다.
4. **탐지 규칙 평가:** 공급량 불일치, 비정상 민팅, 프로젝트 지갑 순유출, DEX 집중 매도 조건을 평가합니다.
5. **결과 표시:** 대시보드에 상태와 근거 데이터를 제시합니다.
6. **온체인 검증:** Explorer에서 컨트랙트·이벤트·트랜잭션을 확인합니다.

## 주요 입력

| 입력 | 역할 |
|---|---|
| 토큰 컨트랙트 | 총공급량, 권한, 전송·민팅 이벤트 확인 |
| 프로젝트 관련 지갑 | 유입·유출과 DEX 상호작용 확인 |
| 공시 또는 등록 공급량 | 온체인 공급량의 비교 기준 |
| DEX 풀·Router·Factory | 스왑 경로와 유동성 환경 확인 |
| 임계값·시간창 | 신호 민감도 설정 |

## 출력의 의미

대시보드의 상태는 설정된 데이터와 규칙이 만든 **검토 우선순위 신호**입니다. 신호는 사건의 의도나 법적 성격을 판정하지 않습니다. 운영자는 관련 주소, 거래 목적, 프로젝트 공시, 시장 상황과 원시 트랜잭션을 함께 검토해야 합니다.

## 현재 데모와 목표 아키텍처

본 데모의 값은 예시입니다. 현재 데모는 RISK 테스트 토큰을 중심으로 데이터 수집과 네 가지 탐지 흐름을 보여줍니다. 향후에는 동일한 데이터 모델과 검증 흐름을 프로젝트 토큰, RWA, 스테이블코인, DeFi 프로토콜 등으로 확장하는 것을 목표로 합니다.
